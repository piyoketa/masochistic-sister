# 戦闘ロジックとフロントとのつなぎ込み

- 戦闘ロジックはBattleが管理し、Battleが計算した盤面の結果をViewに反映する役割のクラスとして、ViewManagerを作成する。フロントの表示は `src/views/EncounterView.vue` （BattleView.vueに命名変更）が持つ。
    - Model-View-ViewModel (MVVM、モデル・ビュー・ビューモデル) アーキテクチャで例えると、以下の関係になる。
        - Model：Battle
        - View：BattleView.vue
        - ViewModel：ViewManager
- BattleからViewManagerへの連動は、以下のような流れになる。
- 例：プレイヤーが自分のターンを終了した時
    - Battleは、次のプレイヤーの操作（次のターンのカード発動）までに盤面に生じることを演算し、 `ActionLog` に積む。
        - なお、途中でプレイヤーのＨＰが0になり、ゲームオーバーが確定した場合は、そこでGameOverを示すActionLogEntityを積む。
    - `ActionLog` には、敵１の行動、敵２の行動、敵３の行動、敵４の行動、ターン開始時のドロー、などが含まれる。
    - ViewManagerは、ActionLogを受け取り、上から一つずつアニメーションをつけながら、盤面に反映していく。
        - アニメーション中は、Battleへの通知が必要となる（＝盤面に影響を与える）プレイヤー操作は、ViewManager側でQueueし、Battleへは通知しない。
    - 例：オークが技「ビルドアップ」を使ったActionLogを受け取った場合
        - オークのComponentを行動中ステータスに変更（背景色を変える）
        - ビルドアップ発動の効果音の再生を開始
        - オークのComponentをBattleから受け取った「ビルドアップ」発動後の状態に更新。具体的には、Stateが描画される
        - 0.3秒ほど待機する
        - オークのComponentを待機中ステータスに変更（背景色を変える）
    - View側で、ActionLogの再生速度を変更したり、アニメーションをスキップしたりすることができる。
        - 再生速度を変更した場合は、各種アニメーションのdurationと待機時間がその分短くなる。
        - スキップした場合は、即座に特定のIndexのActionLogまで進み、その盤面の状態までアニメーション抜きで描画を更新する。
    - １つのActionLogの反映が完了したら、次のActionLogの反映に進み、全てのActionLogの反映が完了したら、Battleへの通知が必要となる（＝盤面に影響を与える）プレイヤー操作をBattleへ通知するのを再開する。
        - ※特定のカードの解説を表示する、ポーズ画面を表示するなど、Battleを変更しない操作は無関係。

# UI

- CardTagやStateは、タイトルのみを表示し、カーソルをホバーした時に、その説明文がオーバーレイウィンドウとして表示される形式にする。

# アニメーション

- なお、プレイヤーや敵キャラクターの「被ダメージ」の演出は、一旦以下とする。
    - キャラクターの右上にオーバーレイで、ダメージ量（例えば、２０ダメージ×３回の連続攻撃なら２０）の赤い数字を表示する。
    - この数字は、アニメーションなしで一気に表示された後、1秒ほどかけて、上方向に移動しながらフェードアウトしていく。
    - 攻撃回数（例えば、２０ダメージ×３回の連続攻撃なら３回）だけ、この表示を行う。この表示の間には0.3秒程度の間隔を開ける。
